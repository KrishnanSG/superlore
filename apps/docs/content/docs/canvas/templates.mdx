A template is a ready, **designed** structure you fill in. Set **`template`** and the frames, icons,
colours and layout appear; you add nodes into the **named frames** (`group: "strengths"`). No
coordinates, no styling — beautiful and correct out of the box, and **an agent reproduces it
reliably every time.**

## Component-shaped templates

Some "templates" aren't whiteboards — they're a superlore **component** with a preset. Use the
component directly:

- **Sprint board · sales pipeline · hiring pipeline · retro** → the
  **[Board](/docs/components/board)** (columns of typed cards).
- **RICE · ICE · weighted decision matrix · RACI** → the **[Comparison](/docs/components/comparison)**
  (options × criteria, every cell typed and queryable).
- **Roadmap · release timeline** → the **[Timeline](/docs/components/timeline)**.

Pick the surface that fits — Canvas for spatial/structural thinking, a component for tabular data.
Everything serialises to the same typed knowledge graph for the MCP.
