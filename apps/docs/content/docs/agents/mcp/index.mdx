superlore ships a first-class **MCP server** over the _same_ structured content the site renders. On
this deploy it is **public — no auth** — and mounted at:

```
/api/mcp
```

<Note title="Same source, two faces">
  The MCP is not a scrape of these pages. It serves the structured index the site is built from, so
  what an agent reads and what a human sees can never drift.
</Note>

## Connecting

Point any MCP-capable client at the endpoint. For Claude-style config:

```json
{
  "mcpServers": {
    "superlore-docs": {
      "type": "http",
      "url": "https://YOUR-DEPLOY/api/mcp"
    }
  }
}
```

Then ask your agent things like _"search the superlore docs for the dual-representation contract"_ or
_"get_page /docs/architecture"_ and it will read this knowledge base directly.

<Tip title="Inheriting auth">
  If a deploy gates the site (Auth.js / SSO), the MCP inherits the same policy. This docs deploy is
  intentionally open so anyone can connect.
</Tip>
