This page is written for **agents** (and the humans directing them) producing superlore content. If
you are an agent asked to "write the docs," follow this.

## The superlore format

A superlore document is **MDX**: a YAML frontmatter block, then Markdown prose enriched with superlore
components. Same format whether a human or an agent writes it.

```mdx
---
title: Page title
summary: One-line, plain-text gloss used for search and the MCP.
tags: [topic, area]
---

Prose, then components.
```

## The one rule: think in data

superlore components are **dual-representation** — each renders for humans and serializes to a typed
knowledge face for the MCP. So author the _data_, not a picture:

<Check title="Do">
  Use `<Timeline items={[...]} />`, `<EntityCard fields={[...]} />`, `<DataTable rows={[...]} />`.
  The agent reading your KB later gets clean records.
</Check>
<Danger title="Don't">
  Don't paste a screenshot of a roadmap, or describe a table in prose. That data is then invisible
  to agents and unsearchable.
</Danger>

## Best practices

<Steps>
  <Step title="Lead with a summary">
    Give every page a `summary`. It powers search ranking and the snippet an agent sees first.
  </Step>
  <Step title="Prefer structural components over prose tables">
    If you're about to write a Markdown table or a bulleted list of facts, reach for `DataTable` or
    `KeyFacts` so the data survives into the knowledge face.
  </Step>
  <Step title="Use refs to connect things">
    Link entities and pages with relations (`depends-on`, `part-of`, `related`). The MCP's
    `navigate` tool follows them.
  </Step>
  <Step title="Keep ids stable">
    Titles become ids. Renaming a heading changes its address — keep them stable so links hold.
  </Step>
  <Step title="One accent, no emoji">
    superlore core UI uses a single violet accent and no emoji. Let structure carry hierarchy.
  </Step>
</Steps>

## Verify with the MCP

After authoring, point an agent at [the MCP](/docs/agents/mcp) and confirm `search` finds your page and
`get_page` returns the structure you intended. If the data isn't there, you wrote a picture.
