superlore is **MCP-native from the ground up** — agents are a first-class audience, not an
afterthought. The same structured corpus the human site renders is served to agents over a
first-class **MCP server**, and superlore ships guidance (and skills) so agents can _author_ KBs
that read cleanly back over that same MCP.

<Note title="One corpus, two readers">
  The MCP is not a scrape of these pages. It serves the structured index the site is built from, so
  what an agent reads and what a human sees can never drift. See [Architecture](/docs/architecture).
</Note>
