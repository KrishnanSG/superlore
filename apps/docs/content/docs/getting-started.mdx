The fastest way to a superlore knowledge base is to **let Claude build it.** Copy two lines, paste
them into Claude, and you get a complete, MCP-enabled KB — no setup, no framework to learn.

## Let Claude build it

<Steps>
  <Step title="Add the superlore plugin (one time)">
    Paste this into Claude:

    ```text
    /plugin marketplace add KrishnanSG/superlore
    /plugin install superlore@superlore
    ```
  </Step>
  <Step title="Ask for your knowledge base">
    In a fresh chat, paste:

    ```text
    Make me a docs site with superlore
    ```

    That's it. Claude takes it from there.
  </Step>
</Steps>

<KeyFacts items={[{ label: "Copy → paste into Claude", value: "Make me a docs site with superlore" }]} />

### What Claude does — automatically

<Steps>
  <Step title="Asks two quick questions">
    **What kind of KB** — a **Company KB** (internal, for your teams + agents) or **Product
    Documentation** (public, for your users) — and **who can read it** — **auth-protected**
    (Google SSO gates the site _and_ the MCP) or **public**.
  </Step>
  <Step title="Warns if a company KB would be public">
    Choose a Company KB without auth and Claude flags it — an internal KB shouldn't be
    world-readable, and the MCP would expose the same content — then offers to add SSO first.
  </Step>
  <Step title="Scaffolds, seeds, and connects">
    It generates the whole project, writes `superlore.json`, seeds a page or two so it isn't empty,
    mounts the MCP route, and offers to register the MCP with your agent — narrating each step.
  </Step>
</Steps>

<Tip title="Then just keep talking to Claude">
  _"add a page about our onboarding"_, _"make a Q3 roadmap"_, _"diagram our architecture"_ — the
  superlore skills turn intent into structured pages. When you're ready, ask Claude to deploy
  (self-host anywhere today; **superlore Cloud** is [waitlisted](/cloud)).
</Tip>

## Or scaffold from the terminal

Prefer a CLI? One command installs it — nothing to clone:

```bash
curl -fsSL https://superlore.vercel.app/install.sh | sh
```

<Note>
  Windows (PowerShell): `irm https://superlore.vercel.app/install.ps1 | iex`. Already have a Node
  package manager? `npm i -g superlore-cli` works too.
</Note>

```bash
superlore init my-kb   # asks the same 2 questions, writes superlore.json, seeds pages
cd my-kb
superlore dev          # local preview at http://localhost:3000
```

`superlore init` finishes by offering to set up your **editor** — it detects VS Code, Cursor, and
Windsurf and installs the **superlore Preview** extension into each, so you get a live,
dual-representation preview of every component as you type. Run it any time on its own:

```bash
superlore connect      # detects your editor(s), installs the live-preview extension
```

<Note>
  One get-started path: install the CLI → `superlore init` scaffolds the KB and sets up your editor →
  then connect the MCP so your agent reads the same corpus (`connect` prints the exact `claude mcp
  add` command, or just ask Claude to _"connect my superlore MCP"_).
</Note>

## Author a first page

```mdx
---
title: Q3 Roadmap
summary: What ships this quarter and when.
tags: [roadmap]
---

<Timeline
  items={[
    { date: "2026-07-01", title: "Kickoff", status: "done" },
    { date: "2026-Q3", title: "GA launch", status: "planned" },
  ]}
/>
```

That single `Timeline` renders a polished, accessible list for humans **and** serializes to a
structured `{ kind: "timeline", items: [...] }` your agent reads over the MCP. You wrote the data
once — there's one source.
